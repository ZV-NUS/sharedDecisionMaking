# 给 Codex 的执行指令

请先阅读并严格执行 `IEEE_Trans_实验出图任务说明_最新版.md`，尤其注意第 7、17、18、20 节。

当前生成的轨迹图存在明显问题：图例遮挡核心区域，无法清楚对比驾驶人意图、机器意图、共享参考轨迹和最终执行轨迹，也无法体现不同方法之间的差异。因此不要只做图形美化，而要重新设计出图逻辑和绘图代码。

请按以下步骤执行：

1. 检查当前项目中已有的数据文件、绘图脚本和输出图片，先不要直接覆盖原图。
2. 新建输出目录 `Experiment_Figures/`，并按照 `.md` 中的文件夹结构保存所有新图。
3. 将 Case 1 的轨迹图至少拆成三张独立图片：
   - `Case1_HM_Intention_Conflict`
   - `Case1_Shared_Reference_Generation`
   - `Case1_Method_Execution_Comparison`
4. 图例必须放在图外或图内空白区域，不能遮挡道路、车辆、轨迹、障碍车、冲突区域和关键时间点。
5. 每张图只表达一个核心结论，不要把人机意图、共享参考和方法对比全部塞在一张图里。
6. 轨迹图必须明确体现：
   - 人机冲突在哪里；
   - 共享决策融合了什么；
   - 本文方法相比其他方法好在哪里。
7. 轨迹图要覆盖完整关键机动过程，包括冲突开始、权限调整、共享参考生成、控制执行和冲突解除。
8. 使用浅灰道路背景和浅灰车道线，突出轨迹，不要使用过深的道路背景。
9. 加入关键时间标注、冲突区域标注、车辆运动方向箭头和车辆快照透明度递进。
10. 所有图采用 IEEE Transactions 风格：Times New Roman 字体、清晰线宽、规范坐标轴、单位和图例。
11. 每个子图单独保存，不要汇总成大图。
12. 每张图同时导出 `.pdf` 或 `.eps` 矢量图，以及 600 dpi `.png` 预览图。
13. 生成后进行自检，逐张检查：
    - 图例是否遮挡；
    - 轨迹是否清晰；
    - 人机意图差异是否明显；
    - 不同方法差异是否明显；
    - 是否能够支撑论文创新点。
14. 最后输出一个 `figure_generation_report.md`，列出：
    - 每张图的文件路径；
    - 使用的数据变量；
    - 该图支撑的论文结论；
    - 是否通过图例遮挡和清晰度自检；
    - 仍需人工确认的问题。

请先给出你准备修改或新增的绘图脚本清单，再执行绘图代码修改。不要删除原始数据和原始图。
